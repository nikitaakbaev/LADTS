# Промт для Claude Code

Ты выступаешь в роли опытного инженера-разработчика систем цифровых двойников (Digital Twin), специалиста по Python, IoT, MQTT, математическому моделированию и 3D-визуализации.

Тебе необходимо разработать полноценный прототип цифрового двойника линейного электромеханического привода с мониторингом состояния в реальном времени.

## Общая концепция проекта

Система должна состоять из трех основных компонентов:

### 1. Physical Twin (физический двойник)

Так как реального привода нет, необходимо реализовать высокореалистичный программный симулятор линейного электромеханического привода.

Симулятор должен использовать математическую модель движения привода, основанную на физических законах.

Использовать модель с учетом:

* положения каретки x(t);
* скорости v(t);
* ускорения a(t);
* силы привода;
* массы подвижной части;
* трения;
* сопротивления среды;
* инерции системы;
* температуры двигателя;
* потребляемого тока.

Реализовать систему дифференциальных уравнений, описывающую движение:

```
m * d²x/dt² = F_motor - F_friction - F_load
```

где:

* F_motor зависит от управляющего сигнала;
* F_friction может включать сухое и вязкое трение;
* F_load — случайная изменяемая нагрузка.

---

### Температурная модель

Добавить тепловую модель:

```
dT/dt = α * I² - β * (T - T_env)
```

где:

* T — температура двигателя;
* I — ток двигателя;
* α — коэффициент нагрева;
* β — коэффициент охлаждения.

---

### Модель тока

Рассчитать ток двигателя как функцию нагрузки:

```
I = k1 * |F_motor| + k2 * |F_load| + шум
```

Добавить реалистичные эффекты:

* случайный шум датчиков;
* небольшую задержку измерений;
* колебания нагрузки;
* дрейф показаний датчиков.

---

### Генерация телеметрии

Симулятор должен работать в реальном времени с частотой не менее 30–60 Гц.

Каждый цикл необходимо генерировать:

```json
{
    "timestamp": 1234567890,
    "position": 0.135,
    "velocity": 0.025,
    "acceleration": 0.003,
    "current": 2.1,
    "temperature": 43.5,
    "health": "NORMAL"
}
```

Статус состояния определять по правилам:

* NORMAL — параметры в норме;
* WARNING — температура или ток приближаются к критическим;
* ERROR — превышены допустимые значения.

---

# 2. Digital Thread

Реализовать передачу данных через MQTT.

Использовать брокер Mosquitto.

Создать следующую структуру топиков:

```
digital_twin/actuator/telemetry
digital_twin/actuator/command
digital_twin/actuator/status
```

---

Симулятор должен быть MQTT Publisher.

Виртуальная модель должна быть MQTT Subscriber.

Использовать JSON для передачи сообщений.

---

# 3. Virtual Twin

Создать виртуальный двойник.

Предпочтительная технология:

## Вариант 1 (рекомендуемый):

Three.js + WebSocket MQTT клиент.

Либо:

* Unity;
* Godot.

---

Визуализация должна содержать:

* 3D модель линейного привода;
* неподвижное основание;
* движущуюся каретку;
* отображение текущего положения.

Движение каретки должно плавно повторять данные:

```
position → координата модели по оси X
```

---

### Визуальная диагностика состояния

Цвет модели должен изменяться:

Зеленый:

```
NORMAL
```

Желтый:

```
WARNING
```

Красный:

```
ERROR
```

Также добавить отображение:

* текущего тока;
* температуры;
* скорости;
* графика изменения параметров во времени.

# Требования к реализации

Использовать:

* Python 3.11+ для симулятора;
* asyncio для циклов реального времени;
* paho-mqtt для MQTT;
* NumPy и SciPy для математического моделирования;
* Three.js для 3D-визуализации;
* Chart.js для графиков.

* Не создавать огромные файлы по 1000 строк.

* Использовать типизацию Python.
* Добавлять комментарии только там, где логика сложная.

Начни работу с Этапа 1 — анализа системы и проектирования архитектуры. Не переходи к следующему этапу без завершения текущего.
