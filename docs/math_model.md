# Mathematical model

## 1. Mechanics

Одномерное движение каретки массой `m`:

```
m · dv/dt = F_motor − F_friction(v) − F_load
dx/dt     = v
```

Trение состоит из кулоновской и вязкой составляющих, сглажено `tanh`,
чтобы избежать дребезга у нуля:

```
F_friction(v) = F_c · tanh(v / v_s) + b · v
```

| Параметр | Значение         | Смысл                               |
|----------|------------------|--------------------------------------|
| `m`      | 2.5 кг           | масса подвижной части               |
| `F_c`    | 3.0 Н            | кулоновское (сухое) трение          |
| `b`      | 8.0 Н·с/м        | коэффициент вязкого трения          |
| `v_s`    | 0.01 м/с         | масштаб сглаживания знака скорости  |
| `F_max`  | 60 Н             | физический предел силы мотора       |
| `[x_min, x_max]` | 0…0.30 м | механические упоры                  |

Интегратор — RK4 c шагом 5 мс. При шаге уход из упоров обнуляет скорость
(жёсткий контакт).

## 2. Thermal model

Сосредоточенная первого порядка:

```
dT/dt = α · I² − β · (T − T_env)
```

| Параметр | Значение            | Смысл                       |
|----------|---------------------|------------------------------|
| `α`      | 0.6 K/(с·А²)        | джоулев нагрев               |
| `β`      | 0.02 1/с            | ньютоново охлаждение         |
| `T_env`  | 25 °C               | температура окружения        |

Стационарная температура при постоянном токе:

```
T_ss = T_env + (α/β) · I²
```

Например, при `I = 1.5 А` → `T_ss ≈ 92 °C`, при `I = 3 А` → `T_ss ≈ 295 °C`
(легко выводит в `ERROR` для проверки).

## 3. Current model

Алгебраическая модель: ток дороже всего обходится силе мотора, плюс
небольшая зависимость от нагрузки + ток покоя.

```
I = k1 · |F_motor| + k2 · |F_load| + I_idle,  clamp to I_max
```

| Параметр | Значение | Смысл                         |
|----------|----------|-------------------------------|
| `k1`     | 0.05 А/Н | коэффициент по силе мотора    |
| `k2`     | 0.02 А/Н | коэффициент по нагрузке       |
| `I_idle` | 0.2 А    | ток покоя                     |
| `I_max`  | 8 А      | физический предел (предохр.)  |

## 4. Load model (Ornstein–Uhlenbeck + spikes)

Нагрузка коррелирована во времени, не белый шум:

```
F_load[k+1] = F_load[k] + θ · (μ − F_load[k]) · dt
                       + σ · √dt · N(0,1)
                       + spike (редкий импульс с экспоненциальным затуханием)
```

| Параметр  | Значение      | Смысл                          |
|-----------|---------------|---------------------------------|
| `μ`       | 4 Н           | средняя нагрузка                |
| `θ`       | 0.5 1/с       | скорость возврата к среднему    |
| `σ`       | 2 Н/√с        | диффузия                        |
| `p_spike` | 0.002 /шаг    | вероятность импульса            |
| `Spike`   | ±15 Н         | амплитуда импульса              |
| `F_load_max` | 40 Н       | абсолютное насыщение            |

## 5. Sensor effects

Поверх «истинного» состояния:

* Аддитивный гауссов шум: `σ_x = 0.2 мм`, `σ_v = 1 мм/с`, `σ_I = 50 мА`,
  `σ_T = 0.1 °C`, `σ_a = 0.02 м/с²`.
* Линейный дрейф температуры (`0.5 м°C/с`) и тока (`0.2 мА/с`) —
  имитирует bias instability реального датчика.
* Задержка измерений — кольцевой буфер длиной 2 шага (~10 мс).

## 6. Health classification

Пороговые правила с гистерезисом, итог — максимум серьёзности по двум
каналам:

| Канал       | WARNING | ERROR  | Гистерезис |
|-------------|---------|--------|------------|
| Temperature | 70 °C   | 90 °C  | 3 °C       |
| Current     | 4 А     | 6.5 А  | 0.3 А      |

Гистерезис не даёт дрожать между уровнями около границы: выход из
ERROR требует упасть ниже `err − hysteresis`, выход из WARNING — ниже
`warn − hysteresis`.

## 7. Controller

PID позиционного контура с derivative-on-measurement (anti-kick) и
ограничением интегральной составляющей (anti-windup):

```
u = Kp · e + Ki · ∫e dt − Kd · v
e = target − x
|∫e dt| ≤ integral_clamp
```

`Kp = 1200, Ki = 30, Kd = 90, integral_clamp = 5`. При `emergency_stop`
выход обнуляется и интеграл сбрасывается.
