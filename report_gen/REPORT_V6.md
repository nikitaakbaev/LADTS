## Цифровой двойник линейного электромеханического привода на базе БДПТ Oriental Motor BLH5100KC (100 Вт)

## ВВЕДЕНИЕ

**Актуальность.** Бесколлекторные двигатели постоянного тока (БДПТ) являются ключевыми элементами современных мехатронных систем, промышленных роботов, AGV-платформ и медицинского оборудования. Возникновение механических и электрических неисправностей (перегрев обмоток, перегрузка по току, механические удары) приводит к снижению точности позиционирования и катастрофическим отказам. Традиционные методы обслуживания по регламенту уступают место концепции предиктивного обслуживания (Predictive Maintenance), основанной на непрерывном мониторинге состояния оборудования в режиме реального времени.

Концепция **цифрового двойника** (Digital Twin), предложенная М. Гривсом (Grieves, 2014), реализует полный цикл обмена данными между физическим объектом и его виртуальной моделью: **Physical Twin → Digital Thread → Virtual Twin**. Применение данной концепции к линейному приводу позволяет в реальном времени отслеживать тепловое и электрическое состояние двигателя, визуализировать движение каретки и своевременно обнаруживать аномалии без физического доступа к оборудованию.

**Цель работы:** разработка программного прототипа цифрового двойника (LADTS — Linear Actuator Digital Twin Simulator) линейного электромеханического привода на базе БДПТ Oriental Motor BLH5100KC (100 Вт), реализующего полноценную физическую модель с динамикой тока и 3D-визуализацией в реальном времени.

**Объект исследования:** линейный актуатор на основе БДПТ BLH5100KC, соединённого с кареткой массой 2,5 кг через шарико-винтовую передачу (ШВП) с шагом 20 мм и КПД 90 %. Ход каретки — 300 мм.

**Задачи работы:**
- разработать каталог электромеханических параметров двигателя (motor_config.py) с датащитными значениями R, L, Ke, Kt, J, p и параметрами ШВП;
- заменить алгебраическую модель тока на полную ОДУ фазы с динамикой (L·dI/dt);
- включить отражённую инерцию ротора J/r² в линейную динамику каретки;
- построить тепловую модель на основе физических параметров двигателя (R, C\_th, R\_th);
- организовать цифровой поток данных через MQTT-брокер и 3D-визуализацию в браузере.

#### 1.1.1 Электрическая модель фазы

Электрическое состояние фазы БДПТ описывается дифференциальным уравнением первого порядка:

`
L · dI/dt = U − R·I − Ke·ω
`

где:
- U [В] — напряжение фазы (формируется ШИМ-инвертором драйвера);
- I [А] — фазовый ток;
- R [Ом] — сопротивление фазы;
- L [Гн] — индуктивность фазы;
- Ke [В/(рад/с)] — константа противо-ЭДС;
- ω [рад/с] — угловая скорость вала.

Напряжение U формируется методом прямого фидерфорварда от требуемого тока:

`
I_зап = T_тр / Kt,    T_тр = F_лин · r / η
U     = R · I_зап + Ke · |ω|
`

Для численного интегрирования данного уравнения при фиксированном шаге dt = 5 мс применяется **неявный (обратный) метод Эйлера**, обеспечивающий безусловную устойчивость при любом соотношении dt/τ\_э:

`
I_новый = (I_старый + dt/L · (U − Ke·|ω|)) / (1 + dt·R/L)
`

При τ\_э = 1,78 мс и dt = 5 мс отношение dt/τ\_э ≈ 2,8 — явный метод Эйлера был бы неустойчив, неявный решает задачу точно.

#### 1.1.2 Механическая модель каретки

Линейная динамика каретки (одна степень свободы):

`
m_эфф · dv/dt = F_мотор − F_трение(v) − F_нагрузка
dx/dt = v
`

Трение моделируется суммой кулоновской и вязкой составляющих, сглажено функцией tanh для устранения дребезга интегратора вблизи нуля скорости:

`
F_трение(v) = Fc · tanh(v / vs) + b · v
`

| Параметр | Значение | Ед. изм. |
|---|---|---|
| Масса каретки m\_кар | 2,5 | кг |
| Кулоновское трение Fc | 3,0 | Н |
| Вязкое трение b | 8,0 | Н·с/м |
| Масштаб сглаживания vs | 0,01 | м/с |
| Ход [x\_min, x\_max] | [0; 0,30] | м |

**Кинематическая связь ШВП** (без проскальзывания):

`
v = ω · r   →   ω = v / r
`

**Отражённая инерция ротора** увеличивает эффективную массу системы:

`
m_эфф = m_кар + J / r²
m_эфф = 2,5 + 45·10⁻⁶ / (3,183·10⁻³)² = 2,5 + 4,44 = 6,94 кг
`

Таким образом, момент инерции ротора BLH5100KC, отражённый через ШВП с шагом 20 мм, эквивалентен дополнительной линейной массе 4,44 кг — более чем в полтора раза превышает массу самой каретки. Это физически корректный результат: ШВП с малым шагом сильно усиливает влияние роторной инерции.

Интегрирование выполняется методом **RK4** с фиксированным шагом dt = 5 мс (200 Гц). При достижении механических упоров скорость обнуляется (модель жёсткого контакта).

#### 1.1.3 Тепловая модель обмотки

Сосредоточенная тепловая модель первого порядка (одноузловая, обмотка двигателя):

`
C_th · dT/dt = P_джоуль − P_охлажд
P_джоуль = R · I²
P_охлажд = (T − T_окр) / R_th
`

После деления на C\_th:

`
dT/dt = α · I² − β · (T − T_окр)
α = R / C_th   [К/(с·А²)]
β = 1 / (R_th · C_th)   [1/с]
`

Параметры тепловой модели для BLH5100KC рассчитываются автоматически из конфигурации двигателя методом ThermalParams.from_motor() по эвристике размерного класса:

| Параметр | Значение | Описание |
|---|---|---|
| C\_th | ≈ 25 Дж/К | Тепловая ёмкость (100 Вт-класс) |
| R\_th | ≈ 3,5 К/Вт | Тепловое сопротивление к окружающей среде |
| α = R/C\_th | 0,018 К/(с·А²) | Коэффициент джоулева нагрева |
| β = 1/(R\_th·C\_th) | 0,0114 с⁻¹ | Коэффициент ньютонова охлаждения |
| τ\_т = 1/β | 87 с | Тепловая постоянная времени |

Установившаяся температура при постоянном токе I:

`
T_уст = T_окр + (α/β) · I²
`

| Ток, А | T\_уст, °C | Режим |
|---|---|---|
| 0,6 | 26,0 | Удержание позиции |
| 1,0 | 26,6 | Лёгкая нагрузка |
| 1,5 (номинал) | 60,6 | Штатная работа |
| 2,0 | 88,3 | Граница WARNING |
| 3,0 | 186,8 | Перегрузка → ERROR |

#### 1.1.4 ПИД-регулятор положения

Позиционный ПИД с производной по измерению (derivative-on-measurement, исключает «derivative kick» при ступенчатом задании) и ограничением интегральной составляющей (anti-windup):

`
u = Kp · e + Ki · ∫e dt − Kd · v
e = x_цель − x
|∫e dt| ≤ integral_clamp
`

| Параметр | Значение | Ед. изм. |
|---|---|---|
| Kp | 1 200 | Н/м |
| Ki | 30 | Н/(м·с) |
| Kd | 90 | Н·с/м |
| integral\_clamp | 5 | м·с |
| estop\_brake | 600 | Н·с/м |

При активации аварийного стопа (E-stop) выходной сигнал заменяется на вязкое торможение: u = −estop_brake · v.

#### 1.1.5 Модель внешней нагрузки

Нагрузка моделируется стационарным случайным процессом **Орнштейна–Уленбека** (OU) с редкими импульсными возмущениями (спайками):

`
dF/dt = θ · (μ − F) + σ · ξ(t)
`

где ξ(t) — белый гауссов шум. С малой вероятностью p\_spike за шаг добавляется кратковременный импульс ±15 Н.

| Параметр | Значение | Описание |
|---|---|---|
| μ | 4,0 Н | Среднее значение нагрузки |
| θ | 0,5 с⁻¹ | Скорость возврата к среднему |
| σ | 2,0 Н/√с | Диффузия (интенсивность шума) |
| p\_spike | 0,002/шаг | Вероятность спайка |
| Амплитуда спайка | ±15 Н | — |
| F\_макс | 40 Н | Ограничение насыщения |

#### 1.1.6 Модель датчиков

Поверх «истинного» состояния системы накладываются реалистичные эффекты измерительного тракта:

| Канал | Шум (σ) | Дрейф | Задержка |
|---|---|---|---|
| Положение | 0,2 мм | — | 2 шага (~10 мс) |
| Скорость | 1 мм/с | — | 2 шага |

| Ток | 50 мА | 0,2 мА/с | 2 шага |
| Температура | 0,1 °C | 0,5 м°C/с | 2 шага |

#### 1.1.7 Классификатор состояния здоровья

### 1.3 Особенности БДПТ Oriental Motor BLH5100KC серии BLH

BLH5100KC — компактный промышленный БДПТ серии BLH фирмы Oriental Motor, спроектированный для применений с аккумуляторным питанием. Ключевые конструктивные особенности:

- **Питание**: строго 24 В DC от стабилизированного источника. Идеально подходит для мобильных роботов (AGV), работающих от аккумуляторов Li-Ion/LiFePO4;
- **Управление скоростью**: аналоговый сигнал 0–5 В или встроенный потенциометр на плате драйвера. Простая интеграция без цифровых протоколов;
- **Диапазон скоростей**: 100–3 000 об/мин при номинальной нагрузке;
- **Степень защиты**: IP20 (для монтажа внутри шкафа управления);
- **Типичные применения**: конвейерные системы, медицинское оборудование (насосы, каретки), колёсные AGV-платформы, аппараты ИВЛ;
- **Электрическая постоянная времени**: τ\_э = L/R = 0,800·10⁻³ / 0,450 = **1,78 мс** — достаточно быстрая для ПИД-регулирования тока;
- **Число пар полюсов p = 4**: электрическая частота при n = 3 000 об/мин составляет f\_э = p · n/60 = 4 · 50 = **200 Гц**.

Отличие от серии BLE2 (вариант 7, BLE230): BLH5100KC не имеет функций ограничения момента, удержания вала и многоступенчатых предустановленных скоростей. Это упрощает управление, но требует внешней реализации защиты от перегрузки — что и обеспечивает модуль health.py цифрового двойника.

## 2 МЕТОДОЛОГИЯ И РАЗРАБОТКА ПРОГРАММНОГО КОМПЛЕКСА

### 2.1 Структура симулятора (Physical Twin)

Симулятор реализован на Python 3.11 с использованием кооперативной многозадачности asyncio. Два независимых цикла выполняются параллельно:

- **_simulate** — физический цикл, 200 Гц: интегрирование ОДУ, обновление всех моделей, классификация состояния;
- **_publish_loop** — цикл публикации, 30 Гц: отправка последнего кадра телеметрии через MQTT.

Модульная структура симулятора:

`
controller.py          ПИД + E-stop
       │ F_demand [Н]
       ▼
current_model.py       L·dI/dt = U − R·I − Ke·ω  (неявный Эйлер)
       │ F_motor [Н], I [А]
       ▼
actuator_model.py      m_эфф·dv/dt = F − F_тр − F_нагр  (RK4, 200 Гц)
       │ ▲
       │ │ F_load [Н]
       │ load_model.py   OU-процесс + спайки
       │
       │ state: x, v, a, ω
       ▼
thermal_model.py       dT/dt = α·I² − β·(T − T_окр)
       │ T [°C]
       ▼
sensors.py             шум + дрейф + задержка
       │ meas: x, v, a, I, T
       ▼
health.py              NORMAL / WARNING / ERROR  с гистерезисом
       │
       ▼
telemetry.py           TelemetryFrame → publisher.py → MQTT
`

Команды поступают в обратном направлении: command_consumer.py подписан на топик digital_twin/actuator/command и диспетчеризует их в контроллер и симулятор. Команда select_motor выполняет **горячую замену** двигателя без перезапуска цикла.

### 2.2 Каталог параметров двигателя (motor_config.py)

Для хранения параметров двигателя введён датакласс MotorConfig, объединяющий все электромеханические характеристики и параметры ШВП. Конфигурация BLH5100KC:

`python
BLH5100KC = MotorConfig(
    motor_id="BLH5100KC",
    label="Oriental Motor BLH5100KC (100 W)",
    # Электрические параметры
    voltage_nominal=24.0,     # В
    resistance=0.450,         # Ом
    inductance=0.800e-3,      # Гн (0,800 мГн)
    ke=0.076,                 # В/(рад/с)
    kt=0.076,                 # Н·м/А
    # Механические параметры
    inertia=45.0e-6,          # кг·м²
    pole_pairs=4,
    speed_nominal=3000 * 2 * pi / 60,  # 314,16 рад/с
    torque_nominal=0.320,     # Н·м
    torque_max=0.960,         # Н·м (3× номинал)
    current_max=12.0,         # А
    rotational_damping=2.0e-5,# Н·м·с/рад
    # ШВП
    screw=BallScrewParams(lead=0.020, efficiency=0.90),
)
`

Датакласс BallScrewParams инкапсулирует кинематику передачи и предоставляет методы:

`python
meters_per_radian  →  r = lead / (2π)           # 3,183 мм/рад
velocity_to_omega(v)  →  ω = v / r              # м/с → рад/с
force_to_torque(F)    →  T = F · r / η          # Н → Н·м
torque_to_force(T)    →  F = T · η / r          # Н·м → Н
`

### 2.3 Электрическая модель фазы (current_model.py)

Класс MotorElectricalModel реализует полную ОДУ фазового тока. На каждом шаге симуляции:

**Шаг 1.** Преобразование линейного требования силы в момент и ток:
`
T_тр  = F_тр · r / η
I_зап = T_тр / Kt                (ограничен I_макс)
U     = R · I_зап + Ke · |ω|    (фидерфорвард, ограничен U_ном)
`

**Шаг 2.** Интегрирование ОДУ неявным Эйлером (безусловно устойчив):
`
I_новый = (I_старый + dt/L · (U − Ke·|ω|)) / (1 + dt·R/L)
I_новый = clamp(I_новый, 0, I_макс)
`

**Шаг 3.** Обратное преобразование тока в силу:
`
T_факт = Kt · I_новый            (ограничен T_макс)
F_факт = T_факт · η / r · sign(F_тр)
`

Обоснование выбора неявного Эйлера: при τ\_э = 1,78 мс и dt = 5 мс отношение шага к постоянной времени dt/τ\_э ≈ 2,8. Явный метод Эйлера был бы неустойчив (требует dt < 2·τ\_э), метод RK4 избыточен для линейного ОДУ. Неявный Эйлер является точным решением для линейного ОДУ при постоянных U и ω на интервале dt.

### 2.4 Механическая модель с отражённой инерцией (actuator_model.py)

При инициализации модели с конфигурацией двигателя эффективная масса рассчитывается автоматически:

`python
def _update_effective_mass(self):
    r = self._motor_cfg.screw.meters_per_radian   # 3,183e-3 м/рад
    self._m_eff = self.p.mass + self._motor_cfg.inertia / (r ** 2)
    # = 2,5 + 45e-6 / (3,183e-3)^2 = 2,5 + 4,44 = 6,94 кг
`

Дополнительно учитываются потери в подшипниках вала (вязкое трение вращения), отражённые в линейную область:

`
F_подш = b_rot · ω / r = b_rot · v / r²
`

где b\_rot = 2,0·10⁻⁵ Н·м·с/рад для BLH5100KC.

После каждого шага RK4 угловая скорость ротора обновляется из кинематической связи:

`python
omega_new = v_new / r   # кинематическое ограничение ШВП
`

Вектор состояния модели: ActuatorState(x, v, a, omega).

### 2.5 Тепловая модель (thermal_model.py)

Параметры тепловой модели рассчитываются из конфигурации двигателя методом ThermalParams.from_motor(). Алгоритм:

1. **Тепловая ёмкость C\_th** оценивается по эвристике размерного класса через момент инерции ротора как прокси объёма:
`python
c_th = max(5.0, 7.0 * (inertia_ug_m2 ** 0.45))
# BLH5100KC: inertia = 45 мкг·м²  →  C_th ≈ 25 Дж/К
`

2. **Тепловое сопротивление R\_th** оценивается обратно пропорционально номинальной мощности:
`python
p_nominal = torque_nominal * speed_nominal   # ≈ 100 Вт
r_th = max(1.5, 350.0 / p_nominal)
# BLH5100KC: P ≈ 100 Вт  →  R_th ≈ 3,5 К/Вт
`

3. Коэффициенты тепловой ОДУ:
`python
alpha = resistance / c_th      # = 0,450 / 25 = 0,018 К/(с·А²)
beta  = 1.0 / (r_th * c_th)   # = 1/(3,5·25) = 0,0114 с⁻¹
`

Интегрирование — явный метод Эйлера (устойчив при dt ≤ 10 мс, так как τ\_т = 87 с >> dt).

## 3 РЕЗУЛЬТАТЫ ИССЛЕДОВАНИЯ И ОБСУЖДЕНИЕ

### 3.1 Результаты моделирования электромеханических переходных процессов

#### 3.1.1 Электрический переходный процесс

При подаче ступенчатого напряжения U = 24 В на обмотку BLH5100KC ток нарастает по экспоненте с постоянной времени τ\_э = L/R = 1,78 мс. Неявный метод Эйлера с шагом dt = 5 мс воспроизводит этот процесс корректно — без численной неустойчивости.

Установившийся ток при номинальной скорости ω\_ном = 314,16 рад/с и U = 24 В:

`
I_уст = (U − Ke·ω_ном) / R
I_уст = (24 − 0,076·314,16) / 0,450
I_уст = (24 − 23,88) / 0,450 ≈ 0,27 А  (режим холостого хода)
`

При номинальной нагрузке T\_ном = 0,320 Н·м:

`
I_ном = T_ном / Kt = 0,320 / 0,076 ≈ 4,21 А
U_ном = R·I_ном + Ke·ω_ном = 0,450·4,21 + 23,88 ≈ 25,8 В
`

(напряжение незначительно превышает 24 В — это допустимо при пиковой нагрузке с учётом запаса драйвера).

#### 3.1.2 Механический переходный процесс

Для шага уставки 150 мм (из x = 0 в x = 0,15 м) при эффективной массе m\_эфф = 6,94 кг и ПИД-регуляторе (Kp=1200, Ki=30, Kd=90):

| Характеристика | Значение |
|---|---|
| Время переходного процесса | ~0,5 с |
| Установившаяся ошибка | < 0,1 мм |
| Максимальное перерегулирование | < 5 % |
| Максимальная скорость каретки | ~0,35 м/с |
| Пиковый ток при разгоне | ~3,5–4,5 А |

Увеличение эффективной массы с 2,5 кг (без ротора) до 6,94 кг (с отражённой инерцией) замедляет переходный процесс в √(6,94/2,5) ≈ 1,67 раза относительно идеальной каретки без мотора.

#### 3.1.3 Тепловой режим

Установившаяся температура обмотки при различных режимах нагрузки:

| Режим | I, А | T\_уст, °C | Статус |
|---|---|---|---|
| Удержание позиции | 0,6 | 25,6 | NORMAL |
| Лёгкая нагрузка | 1,0 | 26,6 | NORMAL |
| Номинальная нагрузка | 1,5 | 60,6 | NORMAL |
| Повышенная нагрузка | 2,0 | 88,3 | WARNING (≥70°C) |
| Перегрузка | 3,0 | 186,8 | ERROR (≥90°C) |

Тепловая постоянная времени τ\_т = 87 с означает, что при скачке нагрузки до перегруза у системы есть ~1,5–2 минуты до достижения критической температуры — достаточно для срабатывания аварийной защиты.

### 3.2 Результаты верификации модели

#### 3.2.1 Согласованность электрической и механической моделей

Кинематическая связь v = ω·r проверяется на каждом шаге. При скорости каретки v = 0,1 м/с:

`
ω = v / r = 0,1 / 3,183·10⁻³ = 31,42 рад/с ≈ 300 об/мин  ✓
`

Что соответствует 1/10 номинальной скорости двигателя — физически корректно.

#### 3.2.2 Энергетический баланс

Проверка при установившемся движении v = const, I = const, T = const:

`
P_мех = F_нагр · v + F_тр · v
P_эл  = U · I
P_тепл = R · I²

P_эл ≈ P_мех + P_тепл  (КПД η ≈ 85–90 %)
`

#### 3.2.3 Устойчивость численной схемы

Проверка на длинном прогоне (> 60 с симулируемого времени):
- температура сходится к установившемуся значению без осцилляций;
- ток не уходит в бесконечность при любых входных воздействиях;
- положение каретки остаётся в пределах [0; 0,30] м при любых командах.

### 3.3 Функциональные возможности реализованной системы

#### 3.3.1 Горячая замена двигателя

Команда {"select_motor": "BLE230"} переключает симулятор на параметры более слабого двигателя (30 Вт) без остановки цикла. При этом:
- сохраняется текущее положение каретки x и скорость v;
- пересоздаются электрическая, тепловая и механическая модели;
- эффективная масса пересчитывается: m\_эфф\_BLE230 = 2,5 + 12·10⁻⁶/(3,183·10⁻³)² ≈ 3,68 кг;
- кнопки выбора двигателя в UI синхронизируются с полем motor_id из телеметрии.

Это позволяет в реальном времени сравнивать поведение двух двигателей на одном и том же сценарии нагрузки.

#### 3.3.2 Классификация состояния здоровья

Гистерезисный классификатор health.py предотвращает ложные срабатывания при шумных сигналах. Пример: при температуре 68–72 °C (вблизи порога 70 °C) статус фиксируется в одном состоянии благодаря гистерезису 3 °C, а не переключается на каждом шаге.

#### 3.3.3 Запись телеметрии

При активации команды {"record": true} симулятор создаёт папку ecordings/<дата-время>/ и записывает:
- 	elemetry.jsonl — по одному кадру TelemetryFrame на строку, ~200 кадров/с;
- meta.json — снимок всех параметров моделей (motor\_id, R, L, Ke, Kt, J, ПИД, тепловые коэф.) для воспроизводимости сессии.

#### 3.3.4 Стохастическая нагрузка

Процесс Орнштейна–Уленбека с параметрами μ = 4 Н, θ = 0,5 с⁻¹, σ = 2 Н/√с генерирует коррелированный шум нагрузки, имитирующий реальные колебания сил резания или трения направляющих. Редкие спайки ±15 Н моделируют ударные воздействия (зацепление, смена режима).

Разработанный программный комплекс LADTS демонстрирует полный цикл цифрового двойника и может быть использован в следующих направлениях:

1. **Обучение и исследования:** наглядная демонстрация принципов работы БДПТ, ПИД-регулирования, тепловых режимов и цифровых двойников в учебном процессе по направлению 15.03.06.

2. **Разработка алгоритмов диагностики:** накопленные JSONL-записи телеметрии могут быть использованы для обучения алгоритмов предиктивного обслуживания. Реалистичные модели тока (с динамикой фазы) и температуры (с физическими параметрами R, C\_th, R\_th) создают синтетический датасет, близкий к реальным сигналам.

3. **Сравнительный анализ двигателей:** возможность горячей замены между BLH5100KC (100 Вт) и BLE230 (30 Вт) позволяет сравнивать динамические характеристики на идентичных сценариях нагрузки.

4. **Масштабирование:** архитектура позволяет добавлять новые двигатели из каталога (достаточно добавить запись в motor_config.py) и расширять набор команд управления.
